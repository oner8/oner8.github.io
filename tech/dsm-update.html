DSM 5 2 升级 6 0 2 amp 安装迅雷远程下载成功

发布于 2016-12-02 tech  群晖 迅雷远程下载 

DSM 6.0 出来好久了，升级强迫症终于忍不住了。

先来张安装成功图。

迅雷远程下载安装开始：

在DSM5.4以前，DSM系统原生支持迅雷远程下载，这给用户提供了相当大的便利，直接将视频下载到NAS系统中，然后在家里任何一台联网设备上直接播放。但是出于某些原因，迅雷跟群辉的合作关系终止，所以DSM5.4开始即不在支持迅雷远程下载，相应软件也遭下架。网上有不少如果从终端切换到Linux root权限下修改相应配置文件，安装软件，最终实现支持迅雷远程下载的教程。但是这些步骤相当繁琐，对Linux没有一定了解的用户可能即使面对一篇有详细截图的教程也不知道在干嘛。就算是Linux老司机看了也觉得有点烦琐。但是这些都不要紧，现在这里有一个更简单的方式。

1）.首先打开DSM的ssh，控制面板 -> 终端机和SNMP -> 启动SSH功能。 然后ssh过去直接docker pull了。

sudo docker pull index.tenxcloud.com/soartou/docker-xware

2）.这个时候再回到 DSM 中，Docker 映像下就可以看到已经有xware的映像了。

3）.选中容器，然后点击“启动”将弹出如下配置框，根据提示为容器取一个合适的名字，然后选中“高级设置”

选择一个文件夹(本示例使用/TDDOWNLOAD)，以及挂载到容器内的路径/TDDOWNLOAD（迅雷远程下载默认是下载到/app/TDDOWNLOAD)

4）.运行容器，查看日志获取迅雷远程“激活码”
运行容器后，可以在容器中查看到正在运行的容器的状态。选择容器，然后点击“详情” –> 日志 查看容器运行过程中的日志。xware中会输出迅雷远程的“激活码“。记住”The Active Code is: 后面的六位字符。

5）.登录迅雷远程并绑定设备
登录迅雷远程，并通过激活码绑定设备，然后开始下载。下载完成后当前面挂载的/TDDOWNLOAD目录下找下载完成的文件。
